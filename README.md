# 🧹 Preprocessor API  

**Module:** News Bias Analysis Pipeline  

---

## 1. Overview

The **Preprocessor API** receives structured JSON output from the News Scraper API and prepares the article content for downstream bias analysis.

Its primary role is to standardize and clean scraped article data so that it becomes consistent, noise-free, and model-ready.

This module acts as the refinement layer between data collection and bias scoring.

---

## 2. Input Source

The API directly consumes the JSON response generated by the News Scraper API.

### Example Input

```json
{
  "title": "Article headline",
  "content": "Raw scraped article text with ads, links, and formatting noise...",
  "source": "News Source Name",
  "url": "https://example.com/article"
}
```

## 3. Processing Strategy

The Preprocessor API applies standardized cleaning functions, with logic that may vary depending on the news source.

Core Cleaning Operations

- Removal of advertisements and promotional sections
- Removal of embedded links and hyperlinks
- Removal of unwanted symbols and formatting artifacts
- Elimination of navigation text and repetitive boilerplate
- Whitespace normalization
- Structural text consolidation

Source-Aware Cleaning

Different news sources structure articles differently.
The API includes source-based cleaning logic to:

- Handle variations in layout
- Remove source-specific unwanted patterns
- Preserve actual journalistic content

## 4. Setup 

### Step 1: Create a Virtual Environment
```bash
python -m venv venv
```

Activate the environment:

Windows:
```
venv\Scripts\activate
```
Mac/Linux:
```
source venv/bin/activate
```
### Step 2: Install Dependencies
```
pip install -r requirements.txt
```


### Step 3: Run the API

If using FastAPI:
```
uvicorn main:app --reload
```
If using Flask:
```
python app.py
```
The API will typically run at:
```
http://127.0.0.1:8000
```

---
## 5. Architecture Role in Pipeline
Input URL
   ↓
News Scraper API
   ↓
Preprocessor API
   ↓
Bias Scoring Engine

The Preprocessor ensures that all articles, regardless of origin, follow a consistent format before entering the bias analysis stage.

## 6. Key Features

- Deterministic cleaning pipeline
- Source-aware standardization
- Modular cleaning functions
- Designed for NLP model compatibility
- Lightweight and extensible architecture
---
7. Maintenance Considerations

- This API requires frequent updates because:
- News sources may change formatting
- Writing styles evolve
- New patterns of embedded content may appear
- Different methods of including advertisements or unrelated content may be introduced
- Continuous monitoring and rule updates are essential to maintain cleaning accuracy.

---
8. Limitations

- Does not perform semantic correction
- Does not summarize content
- Assumes structured input from the scraper
- May require tuning for multilingual content
